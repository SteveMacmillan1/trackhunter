FAQ

General

How does TrackHunter work?

Start with a song you like and we'll give you similar song recommendations to check out. We'll never show you the same recommendation twice, and you can ban artists from appearing in your recommendations feed. Bookmark songs that you like as you go, then export them directly to your Spotify account!

Who is it intended for?

Our primary goal is to help serious music curators find deep tracks with the genres and characteristics they want. But we also have search options to help users discover the bigger hits in genres they're still new to.

Does banning artists affect the music style of my recommendations?

No. We just filter songs from those artists out individually by the artist's Spotify ID.

What was the site built with?

Spotify API Deprecations

What happened?

Spotify API deprecated its /recommendations endpoint in November 2024. This has unfortunately ended our music discovery use case for now but we may revive the project at some time (see "Future Plans" below).

How does that affect things?

TrackHunter now issues the same 50 song recommendations in a loop. They're loaded from a JSON file that mimics a /recommendations response, so the site still handles and displays content dynamically as though it received that data from Spotify.

So what does TrackHunter actually do now, then?

From a user's perspective:

Unfortunately not much. TrackHunter doesn't have realistic use cases when forced to loop the same 50 recommended tracks.

From a developer's pespective:

It still filters songs that have already been viewed, still filters songs from banned artists, still fetches seed tracks directly from the Spotify API, still acquires permissions from Spotify user accounts through OAuth 2.0, and still exports bookmarked tracks to a user's Spotify account. So it's still a full-stack website with database CRUDing that interacts with a 3rd party API.

Any future plans for TrackHunter?

We'd like to implement another song recommendation API service to the backend. The goal would be to make TrackHunter platform-agnostic by using songs' universal ISRC IDs. That way we could export a user's bookmarked songs to any platform that could support importing. We've checked out Deezer, last.fm, and other similar APIs for song recommendations but so far haven't found something that'll quite work at this time.
